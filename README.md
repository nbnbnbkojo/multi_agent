# 多智能体智能问答项目 README.md

# Multi-Agent Intelligent Q&A Project

一款基于旧版LangChain（0.2.x）、Qwen大模型、Milvus向量数据库开发的多智能体智能问答系统，专注于「食品安全法专业问答」与「通用领域问答」双场景，采用纯Python脚本开发（无Web依赖），支持非流式/流式两种问答模式，架构轻量、部署简单、调用便捷。

## 📋 项目简介

本项目通过多智能体协作实现精准问答，核心分为「领域识别智能体」「RAG检索智能体」「通用搜索智能体」三大核心模块，无需复杂配置，可直接本地运行或集成到其他Python项目中。

核心优势：

- ✅ 领域自动识别：精准区分「食品安全法」与「通用领域」问题，分支处理更高效

- ✅ 双问答模式：支持非流式（直接返回结果）与流式（实时打印）两种调用方式

- ✅ 轻量无依赖：纯Python脚本开发，移除FastAPI等Web依赖，运行更轻便

- ✅ 专业知识库：集成Milvus向量数据库，食品安全法领域基于RAG检索，回答更精准

- ✅ 友好交互：通用领域采用动漫风格语气，回答生动易懂，支持交互式命令行测试

- ✅ 高级检索优化：内置**多路召回 + 重排序（Rerank）混合检索策略**，大幅提升专业问答准确率

## ✨ 优化亮点：多路召回 + 重排序（Rerank）混合检索策略

为解决传统向量检索**召回不全、相关性不准**的问题，本项目在 RAG 模块中正式引入**混合检索引擎**：

1. **多路召回（Multi-Recall）**
   - 向量召回：基于 Milvus 进行语义相似度检索
   - 关键词召回：基于 BM25/规则匹配进行精准匹配
   - 多路结果合并，确保不丢失高价值上下文

2. **重排序（Rerank）**
   - 使用专业重排序模型（如 bge-reranker）
   - 对多路召回结果进行相关性打分
   - 只保留 Top-K 最相关的文档送入大模型

3. **优势**
   - 检索覆盖率提升
   - 回答精准度显著提高
   - 减少幻觉，专业领域问答更可靠

### 各模块功能说明

1. **领域识别智能体（DomainAgent）**：核心负责判断用户提问所属领域，区分「食品安全法」与「通用领域」，为后续问答分支提供依据。

2. **RAG检索智能体（RAGAgent）**：仅处理食品安全法领域问题，采用**多路召回+重排序**混合检索策略，调用Milvus向量数据库 + 关键词检索获取上下文，结合Prompt生成专业、精准、低幻觉的回答。

3. **通用搜索智能体（BoChaAgent）**：处理非食品安全法问题，调用通用搜索接口获取相关信息，结合动漫风格语气生成轻松易懂的回答。

4. **Qwen大模型封装（QwenModel）**：统一封装大模型调用接口，支持非流式（invoke）和流式（stream）两种生成方式，适配不同使用场景。

5. **Milvus连接器（MilvusConnector）**：负责与Milvus向量数据库交互，提供知识库相似检索功能，支撑RAG问答的精准性。
