# Multi-Agent Intelligent Q&A Project

一款基于旧版LangChain（0.2.x）、Qwen大模型、Milvus向量数据库开发的多智能体智能问答系统，专注于「食品安全法专业问答」与「通用领域问答」双场景，采用纯Python脚本开发（无Web依赖），支持非流式/流式两种问答模式，架构轻量、部署简单、调用便捷。

## 📋 项目简介

本项目通过多智能体协作实现精准问答，核心分为「领域识别智能体」「RAG检索智能体」「通用搜索智能体」三大核心模块，无需复杂配置，可直接本地运行或集成到其他Python项目中。

核心优势：

- ✅ 领域自动识别：精准区分「食品安全法」与「通用领域」问题，分支处理更高效

- ✅ 双问答模式：支持非流式（直接返回结果）与流式（实时打印）两种调用方式

- ✅ 轻量无依赖：纯Python脚本开发，移除FastAPI等Web依赖，运行更轻便

- ✅ 专业知识库：集成Milvus向量数据库，食品安全法领域基于RAG检索，回答更精准

- ✅ 友好交互：通用领域采用动漫风格语气，回答生动易懂，支持交互式命令行测试

### 各模块功能说明

1. **领域识别智能体（DomainAgent）**：核心负责判断用户提问所属领域，区分「食品安全法」与「通用领域」，为后续问答分支提供依据。
     

2. **RAG检索智能体（RAGAgent）**：仅处理食品安全法领域问题，调用Milvus向量数据库进行相似检索，获取相关知识库上下文，结合Prompt生成专业回答。
      

3. **通用搜索智能体（BoChaAgent）**：处理非食品安全法问题，调用通用搜索接口获取相关信息，结合动漫风格语气生成轻松易懂的回答。
      

4. **Qwen大模型封装（QwenModel）**：统一封装大模型调用接口，支持非流式（invoke）和流式（stream）两种生成方式，适配不同使用场景。
      

5. **Milvus连接器（MilvusConnector）**：负责与Milvus向量数据库交互，提供知识库相似检索功能，支撑RAG问答的精准性。

### 运行项目

```bash

```

运行后将出现交互式菜单，按提示输入功能编号即可使用：

- 输入「1」：非流式问答（直接返回纯文本回答，最常用）

- 输入「2」：流式问答（控制台实时打印回答，模拟流式效果）

- 输入「q」：退出程序


## ⚠️ 注意事项

1. LangChain版本：本项目基于旧版LangChain（0.2.x）开发，**禁止使用新版LangChain**（新版API不兼容，会导致智能体初始化失败）。

2. API密钥：确保`.env`文件中的DASHSCOPE_API_KEY配置正确，否则无法调用Qwen大模型。

3. Milvus配置：确认Milvus服务已启动，连接信息（host、port）与`milvus_agent.py`中一致，否则RAG检索会失败。

4. 日志查看：问答异常时，可查看`qa_error.log`文件，获取详细错误信息便于排查。

5. 忽略文件：`.gitignore`已配置忽略.idea、venv、日志等非核心文件，无需手动修改。

## 🔧 常见问题排查

- **问题1：运行报错「No module named 'agent.domain_agent'」**
解决方案：检查项目目录结构是否与本文档一致，确保`agent/`目录下存在对应文件，且运行脚本时的工作目录是项目根目录。
      

- **问题2：调用Qwen大模型报错「API key invalid」**
解决方案：检查`.env`文件中的DASHSCOPE_API_KEY是否正确，替换为有效密钥后重新运行。
      

- **问题3：RAG问答无相关上下文，回答不准确**
解决方案：确认Milvus向量数据库中已导入食品安全法知识库，且`milvus_agent.py`中的collection_name与数据库中一致。


- **问题4：流式问答无输出或输出中断**
解决方案：检查Qwen大模型的stream接口是否正常，网络是否稳定，尝试重新运行脚本。
